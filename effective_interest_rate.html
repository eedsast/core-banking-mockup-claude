Effective interest rate

Effective interest rate calculation is based on changing APR ( Annual percetage rate) until Sum of discounted network flows (using that particular APR is equal ZERO.

When we calculate APR, what we’re really doing is discounting future cash flows back to their present value and equating that to the net amount disbursed to the borrower.

·        CFt​ = net cash flow at time t (from the lender’s perspective; disbursement is negative, repayments positive)

·        f = number of periods per year (e.g. 12 for monthly)

·        n = total number of instalments

What is difference between APR and EIR according to European Central Bank and Central Bank of the Republic of Kosovo (CBK)?

According to European Central Bank (ECB) and IFRS guidelines, certain costs and charges are explicitly excluded from the EIR calculation and not treated as interest (like non-compulsory insurance, security deposit, administrative, commitment, processing fees, fee for early repayment).


For OneFor case EIR will always be equal to APR because there is no Fees, Insurance or Security deposits.

Valid code (in PostgreSQL):

-- Recreate cleanly
DROP PROCEDURE IF EXISTS public2.calculate_apr(int);
 
CREATE OR REPLACE PROCEDURE public2.calculate_apr(IN p_loan_account_id int)
LANGUAGE plpgsql
AS $procedure$
DECLARE
    apr numeric;                         -- working APR guess (in %)
    tol CONSTANT numeric := 0.0001;      -- convergence tolerance
    max_iter CONSTANT int := 10000;
    iter int := 0;
    f_apr numeric;
    f_prime_apr numeric;
    df_period numeric;                   -- time in years for each cash flow
    rec public2.loan_repayment_plan%ROWTYPE; -- <-- declare loop variable
BEGIN
    -- Initialize with the loan's nominal interest rate (in %)
    SELECT la.interest_rate
      INTO apr
      FROM public2.loan_account la
     WHERE la.loan_account_id = p_loan_account_id
     LIMIT 1;
 
    IF apr IS NULL THEN
        RAISE EXCEPTION 'Loan % not found or interest_rate is NULL', p_loan_account_id;
    END IF;
 
    -- Newton–Raphson to find APR where NPV ≈ 0
    LOOP
        f_apr := 0;
        f_prime_apr := 0;
 
        FOR rec IN
            SELECT *
              FROM public2.loan_repayment_plan
             WHERE loan_account_id = p_loan_account_id
        LOOP
            df_period := (rec.years_fractals
                          + rec.months_fractals / 12.0
                          + rec.days_fractals::numeric / rec.number_of_days_in_year);
 
            f_apr := f_apr
                   + rec.net_cash_flow
                     / POWER(1 + apr/100.0, df_period);
 
            f_prime_apr := f_prime_apr
                          - rec.net_cash_flow * df_period
                            / POWER(1 + apr/100.0, df_period + 1);
        END LOOP;
 
        EXIT WHEN ABS(f_apr) < tol OR iter >= max_iter;
 
        IF f_prime_apr = 0 THEN
            RAISE NOTICE 'Derivative hit zero; stopping at iter %, apr %', iter, apr;
            EXIT;
        END IF;
 
        apr := apr - (f_apr / f_prime_apr);  -- NR step
        iter := iter + 1;
    END LOOP;
 
    -- Persist per-row discounted NCFs with the final APR
    UPDATE public2.loan_repayment_plan r
       SET discounted_net_cash_flow =
             r.net_cash_flow
             / POWER(1 + apr/100.0,
                     (r.years_fractals
                      + r.months_fractals/12.0
                      + r.days_fractals::numeric / r.number_of_days_in_year))
     WHERE r.loan_account_id = p_loan_account_id;
 
    -- Persist final APR
    UPDATE public2.loan_account
       SET effective_interest_rate = apr
     WHERE loan_account_id = p_loan_account_id;
 
    RAISE NOTICE 'APR for loan_id % is % (iterations: %)', p_loan_account_id, apr, iter;
END;
$procedure$;
 
-- How to run:
-- CALL public2.calculate_apr(12345);
