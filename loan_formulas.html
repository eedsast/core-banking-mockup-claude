Loan Calculation Formulas — Universal Reference

Periodic rate, EMI/annuity, and interest-portion formulas covering all four day-count conventions (30/360, ACT/360, ACT/365, ACT/ACT) and both calculation methods (Conforming, Proportional)

Notation

Symbol Meaning
$P$ Principal (loan amount)
$r$ Annual nominal interest rate (e.g. 0.17)
$n$ Number of instalments
$B_{k-1}$ Outstanding balance at start of period $k$
$D_k$ Days in period $k$ (per day-count convention)
$Y_k$ Days in year for period $k$ (per day-count convention)
$f_k$ Year-fraction (accrual factor) for period $k$ = $D_k / Y_k$
$r_m$ Periodic rate used to compute the annuity
$A$ Equal instalment (EMI)
$I_k$ Interest portion of instalment $k$

1. Periodic Rate $r_m$ (Universal)

Periodic Rate $$r_m = \begin{cases} (1 + r)^{f} - 1 & \text{if CONFORMING} \\[6pt] r \times f & \text{if PROPORTIONAL} \end{cases}$$

Where $f$ is the year-fraction, resolved by the day-count convention:

Year-Fraction by Day-Count $$f = \frac{D}{Y} = \begin{cases} \dfrac{30}{360} = \dfrac{1}{12} & \text{30/360} \\[8pt] \dfrac{\text{ActualDays}}{360} & \text{ACT/360} \\[8pt] \dfrac{\text{ActualDays}}{365} & \text{ACT/365} \\[8pt] \dfrac{\text{ActualDays}}{Y_{\text{actual}}} & \text{ACT/ACT (split at year-end)} \end{cases}$$

For the annuity calculation, $f$ is the regular-period fraction (the nominal month: 1/12 for monthly). For 30/360 this always equals 1/12; for ACT conventions most core banking systems also use the nominal 1/12 for the EMI base, then apply actual-day fractions per period for the interest split (step 3).

2. EMI / Annuity Calculation (Universal)

Equal Instalment (EMI) $$A = P \times \frac{r_m \times (1 + r_m)^n}{(1 + r_m)^n - 1}$$

This formula is the same for all four day-count conventions and both methods — the only thing that changes is the value of $r_m$ fed into it (from step 1).

Edge case — zero rate
If $r_m = 0$ (interest-free loan), the formula divides by zero. Handle separately: $A = P / n$ when $r_m = 0$.

3. Interest Portion per Period (Universal)

Interest Portion $$I_k = B_{k-1} \times g(r, f_k)$$

Where $g$ applies the same method choice as step 1, but using each period's own actual year-fraction $f_k = D_k / Y_k$:

Method Function $$g(r, f_k) = \begin{cases} (1 + r)^{f_k} - 1 & \text{if CONFORMING} \\[6pt] r \times f_k & \text{if PROPORTIONAL} \end{cases}$$

Then the principal portion and new balance:

$$P_k = A - I_k \qquad B_k = B_{k-1} - P_k$$

ACT/ACT Year-Fraction (the only convention needing a split)

For ACT/ACT, when a period crosses 31-Dec, $f_k$ is the sum of two sub-fractions:

Year-Boundary Split $$f_k = \frac{D_{k,Y_1}}{Y_1} + \frac{D_{k,Y_2}}{Y_2}$$

Where $Y_1, Y_2 \in \{365, 366\}$ depending on whether each calendar year is a leap year. If the period lies within a single year, this reduces to $D_k / Y_k$ with $Y_k = 366$ in a leap year, else 365.

The Critical Consistency Rule

Consistency
The method choice (CONFORMING / PROPORTIONAL) must be identical in step 1 and step 3. Mixing them breaks the schedule — the last instalment won't close the balance to zero. The day-count convention also flows through both steps consistently.

Last Instalment Rule (Balance Closing)

Because of rounding, the final instalment is computed differently to guarantee a clean zero balance:

$$\text{For } k < n: \quad I_k = B_{k-1} \times g(r, f_k), \quad P_k = A - I_k$$ $$\text{For } k = n: \quad I_n = B_{n-1} \times g(r, f_n), \quad P_n = B_{n-1}, \quad A_n = P_n + I_n$$

The last instalment $A_n$ may differ from $A$ by a cent or two — it absorbs all accumulated rounding.

Unified Pseudocode

function buildSchedule(P, r, n, method, dayCount, dates):
    # Step 1 — periodic rate for annuity (nominal regular-period fraction)
    f_regular = regularFraction(dayCount)        # 1/12 for monthly
    rm = periodicRate(method, r, f_regular)

    # Step 2 — annuity (EMI)
    if rm == 0:
        A = round(P / n, 2)
    else:
        A = round(P * (rm * (1+rm)^n) / ((1+rm)^n - 1), 2)

    # Step 3 — per-period interest split
    balance = P
    for k in 1..n:
        f_k = yearFraction(dayCount, dates[k-1], dates[k])   # actual days / Y
        periodRate_k = applyMethod(method, r, f_k)
        interest = round(balance * periodRate_k, 2)
        if k == n:
            principal = round(balance, 2)        # close the balance
            instalment = round(principal + interest, 2)
        else:
            principal = round(A - interest, 2)
            instalment = A
        balance = round(balance - principal, 2)
        emit(k, dates[k], interest, principal, instalment, balance)

# ── helper functions ──────────────────────────────────────────
function periodicRate(method, r, f):
    if method == CONFORMING: return (1 + r)^f - 1
    else:                    return r * f

function applyMethod(method, r, f):       # identical to periodicRate
    if method == CONFORMING: return (1 + r)^f - 1
    else:                    return r * f

function yearFraction(dayCount, d0, d1):
    switch dayCount:
        case THIRTY_360: return 30 / 360
        case ACT_360:    return actualDays(d0, d1) / 360
        case ACT_365:    return actualDays(d0, d1) / 365
        case ACT_ACT:
            if year(d0) == year(d1):
                Y = isLeap(year(d0)) ? 366 : 365
                return actualDays(d0, d1) / Y
            else:
                ye = endOfYear(d0)                       # Dec 31
                d1part = actualDays(d0, ye)
                d2part = actualDays(ye, d1)
                Y1 = isLeap(year(d0)) ? 366 : 365
                Y2 = isLeap(year(d1)) ? 366 : 365
                return d1part/Y1 + d2part/Y2

  • ACT/360 is the most expensive in both columns — the 360 denominator over-counts time relative to the actual 365/366-day calendar.
  • ACT/ACT is the cheapest here because most periods fall in the 366-day leap year, shrinking each day's fraction.
  • The method gap (Conforming → Proportional) is ~3.5 EUR regardless of day-count — roughly 2× the spread caused by switching day-count (~0.8 EUR between cheapest and dearest within a column).
  • The annuity is identical across day-counts within each method because it is derived from the nominal monthly rate; the day-count only redistributes interest within the schedule and nudges the final instalment to close the balance.
