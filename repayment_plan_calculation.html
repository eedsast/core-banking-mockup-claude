Repayment Plan calculation

User story

As a User
I want to be able to:

  • create Repayment Plan on the fly as kind of what-if analysis (should not be stored anywhere) and

  • create official one during Loan application process what should be stored and connected with Loan application id.

Context

This is the full process picture:

This is proposal of the process that is aligned with our existing process (integrated with Fimple).

During application process we can store Repayment Plan in loan_repayment_plan table.

After initial decisioning and approval we can change the status of Loan application and open Loan in Loan_accounts table in status Approved.

After Digital signature of the contract we can create records in Loan_schedule and Loan_instalment table according to signed contract.

Instalment Date logic:

Loan opened/disbursed on any day in month i.e. 18th Aug 2025 have instalments that are due on: 18th Sep, 18th Oct, 18th Nov, 18th Dec, 18th Jan, 18th Feb, 18th Mar, 18th Apr, 18th May, 18th Jun, 18th Jul, 18th Aug 2026.

Loan opened/disbursed on 29th day in month i.e. 29th Aug 2025 have instalments that are due on: 29th Sep, 29th Oct, 29th Nov, 29th Dec, 29th Jan, 28th Feb (or 29th in leap years), 29th Mar, 29th Apr, 29th May, 29th Jun, 29th Jul, 29th Aug 2026.

Loan opened/disbursed on 30th day in month i.e. 30th Sep 2025 have instalments that are due on: 30th Oct, 30th Nov, 30th Dec, 30th Jan, 28th Feb (or 29th in leap years), 30th Mar, 30th Apr, 30th May, 30th Jun, 30th Jul, 30th Aug 2026, 30th Sep 2026.

Loan opened/disbursed on 31st day in month i.e. 31st Aug 2025 have instalments that are due on: 30th Sep, 31st Oct, 30th Nov, 31st Dec, 31st Jan, 28th Feb (or 29th in leap years), 30th Mar, 30th Apr, 31st May, 30th Jun, 31st Jul, 31st Aug 2026.

  • Years fractals are full fractals of year between instalment date and loan opening date

  • Month fractals are full fractals of month between instalment date and loan opening date

  • Day fractals are full fractals of day between instalment date and loan opening date

Instalment, Principal and Interest calculation logic:

The following calculation logic are applied for every installment:

  • Instalment is always equals rounded EMI amount

  • Calculate Instalment interest (Remaining loan balance * nominal annual rate/12)

  • Calculate Instalment principal (Instalment - Instalment interest)

  • Amounts can be rounded on two digits according two principles defined in Loan Product definition (on the nearest amount - usual one - or on the nearest higher amount)

  • Calculate remaining Loan Balance (Loan Balance = Previous Loan Balance - Instalment principal)

  • NCF - Netto Cash flow is equal to the pay in and pay out amounts

  • DNCF - Discounted netto cash flow is calculated with the apr = nominal interest rate in the first iteration. Later on (during Effective interest calculation) we will change this.

Adjustment of the last instalment:

  • Adjustment of the last instalment is optional (defined in Loan Product definition). For that purpose You have to calculate and cumulate the difference between rounded interest amount and real interest amount (a.k.a. residual amount).

  • If we have Last instalment adjustment flag = True then You have to apply the following logic

the last principal = loan amount - cumulative rounded principal

the last interest = rounded (interest of the last instalment + residual interest amount)

only in the last round instalment is not equal to rounded EMI then to the sum of last principal and the last interest

PSEUDO CODE logic:

  1. Set initial values

  2. Calculate EMI amount

  3. Insert disbursement record

  4. Run n = number instalments loops to:

    1. calculate the next instalment date

    2. calculate days in particular year (applying 366 in the leap year)

    3. calculate full fractals of year, month and day

    4. calculate interest = remaining balance * annual interest / 12

    5. calculate principal = rounded instalment - rounded interest

    6. round of instalments, interest, principal

    7. calculate and cumulate difference between rounded amount and real calculated amount for interest and principal

    8. calculate NCF

    9. calculate DNCF

  5. If the adjustment have to be applied on the last instalment then

    1. the e last principal = loan amount - cumulative rounded principal

    2. the last interest = rounded (interest of the last instalment + residual interest amount)

    3. instalment = last principal + last interest
